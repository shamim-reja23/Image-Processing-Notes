1. Image Processing System Overview

Fig 1.1 — Image Processing Pipeline

A digital image processing system takes a real-world scene, digitizes it, processes it computationally, and extracts meaningful information. The knowledge base guides all stages — it contains domain knowledge like anatomical structures in medical imaging or road rules in autonomous driving.

2. Image Digitization: Sampling & Quantization

Sampling (Spatial)

Converting a continuous spatial signal into a discrete grid of pixels. The number of samples in x and y gives image resolution (e.g., 512×512).

Nyquist Theorem: Sampling rate ≥ 2 × maximum frequency to avoid aliasing.

Quantization (Amplitude)

Converting continuous intensity values into discrete grey levels. With k bits, we get 2k grey levels.

8-bit → 256 levels (0–255). Common in most images.

Fig 2.1 — Sampling & Quantization Process
📐 Numerical — Image Storage Size

Q: An image is 512×512 pixels, 8-bit grayscale. How many bytes does it need?

1
Total pixels = 512 × 512 = 262,144 pixels
2
Bits per pixel = 8 (since 2⁸ = 256 grey levels)
3
Total bits = 262,144 × 8 = 2,097,152 bits
4
Total bytes = 2,097,152 / 8 = 262,144 bytes ≈ 256 KB

For color (RGB): multiply by 3 → 786,432 bytes ≈ 768 KB

3. Digital Image Fundamentals

A digital image is a 2D function f(x, y) where (x,y) are spatial coordinates and f is the intensity. For grayscale: f ∈ [0, 255]. For color: f is a vector [R, G, B].

6. Contrast Intensification

Linear Stretching (Min-Max Normalization)

s = (r - r_min) / (r_max - r_min) × (L - 1) r = input pixel value r_min = minimum intensity in image r_max = maximum intensity L = number of grey levels (256) s = output pixel value
📐 Numerical — Linear Stretching

Q: Image has pixel values in range [50, 180]. Stretch to [0, 255].

1
r_min = 50, r_max = 180, L - 1 = 255
2
Formula: s = (r - 50) / (180 - 50) × 255 = (r - 50) / 130 × 255
3
For r = 50: s = 0 / 130 × 255 = 0
4
For r = 115: s = 65/130 × 255 = 127.5 ≈ 128
5
For r = 180: s = 130/130 × 255 = 255

Non-Linear Stretching (Power Law / Gamma)

s = c × r^γ (Power Law / Gamma Correction) γ < 1 → expands dark regions (brighten dark image) γ > 1 → compresses dark, expands bright regions γ = 1 → identity (no change) c → scaling constant Log Transform: s = c × log(1 + r) → expands dark, compresses bright Negative: s = (L - 1) - r → inverts image

7. Histogram Equalization

A histogram shows the frequency of each grey level in an image. Histogram Equalization redistributes intensities to achieve a uniform (flat) histogram, enhancing global contrast.

Steps: 1. Compute histogram: h(r_k) = count of pixels with intensity r_k 2. Compute PDF: p(r_k) = h(r_k) / (M × N) 3. Compute CDF (Cumulative Distribution Function): CDF(r_k) = Σ p(r_j) for j = 0 to k 4. Equalized output: s_k = round( CDF(r_k) × (L - 1) )

9. Correlation & Convolution

Correlation

Template matching — measures similarity between kernel and image patch. No flipping of kernel.

(f ⋆ g)(x,y) = Σ Σ f(s,t) × g(x+s, y+t)

Convolution

Filtering operation. Kernel is flipped 180° before sliding. Most filter operations use convolution.

(f * g)(x,y) = Σ Σ f(s,t) × g(x-s, y-t)
📐 Numerical — 1D Convolution

f = [1, 2, 3, 4, 5], kernel h = [1, 0, -1] (flipped: [-1, 0, 1])

1
Flip kernel: h_flipped = [-1, 0, 1]
2
Slide and multiply (with zero padding):
Position 2: f[1,2,3] · h[-1,0,1] = -1×1 + 0×2 + 1×3 = 2
Position 3: f[2,3,4] · h[-1,0,1] = -1×2 + 0×3 + 1×4 = 2
Position 4: f[3,4,5] · h[-1,0,1] = -1×3 + 0×4 + 1×5 = 2
3
Output ≈ [−1, 2, 2, 2, 5] → detects linear gradient
Key difference: Convolution = flip kernel then correlate. For symmetric kernels (Gaussian, Box), convolution = correlation. Always flip for non-symmetric kernels (Sobel, Prewitt).

12. Frequency Domain Filtering

Transform image to frequency domain using Fourier Transform, apply filter, transform back.

2D DFT (Discrete Fourier Transform): F(u,v) = Σ Σ f(x,y) × e^{-j2π(ux/M + vy/N)} x y Inverse DFT: f(x,y) = (1/MN) Σ Σ F(u,v) × e^{j2π(ux/M + vy/N)} u v Convolution Theorem: f*h ↔ F·H (convolution in spatial = multiplication in frequency)
Fig 12.1 — Frequency Domain Filters

Butterworth & Gaussian Filters (Frequency Domain)

Butterworth LPF: H(u,v) = 1 / [1 + (D(u,v)/D₀)^(2n)] Gaussian LPF: H(u,v) = e^{-D²(u,v) / (2σ²)} D(u,v) = √[(u - M/2)² + (v - N/2)²] (distance from center) D₀ = cutoff frequency

Butterworth: order n controls sharpness of cutoff. Gaussian: no ringing artifacts (smooth transition).

13. Mathematical Morphology

Erosion (⊖)

Shrinks foreground objects. Removes small noise. A pixel is 1 only if ALL pixels under SE are 1.

A ⊖ B = {z | B_z ⊆ A}

Dilation (⊕)

Expands foreground objects. Fills holes. A pixel is 1 if ANY pixel under SE is 1.

A ⊕ B = {z | B̂_z ∩ A ≠ ∅}

Opening (∘) = Erosion then Dilation

A ∘ B = (A ⊖ B) ⊕ B

Removes small bright objects (noise), preserves overall shape.

Closing (•) = Dilation then Erosion

A • B = (A ⊕ B) ⊖ B

Fills small holes/gaps in objects.

14. GLCM — Grey Level Co-occurrence Matrix

GLCM captures texture by counting how often pairs of pixel intensities occur at a given spatial relationship (distance d, angle θ).

GLCM[i,j] = count of pairs where: pixel at position p has intensity i, pixel at offset (d, θ) has intensity j Common angles: 0° (horizontal), 45°, 90°, 135° Normalize: P(i,j) = GLCM[i,j] / total_pairs

15. Local Binary Pattern (LBP)

LBP is a texture descriptor that encodes local structure by comparing each pixel to its neighbors.

Algorithm: 1. For each pixel p with value g_c: 2. Compare with 8 neighbors: if g_neighbor ≥ g_c → 1, else → 0 3. Read bits clockwise from top-left → 8-bit binary number → decimal = LBP code 4. Build histogram of LBP codes over image → texture descriptor Extended: LBP(P, R) — P neighbors at radius R P = 8, R = 1 → standard 3×3 P = 16, R = 2 → larger neighborhood Uniform LBP: patterns with ≤ 2 transitions (01 or 10 changes) → 58 uniform + 1 non-uniform bin = 59 bins

17. Thresholding

Simplest segmentation technique — convert grayscale to binary by selecting a threshold T.

g(x,y) = 1 if f(x,y) > T (foreground) 0 otherwise (background) Global: T is one value for entire image Local: T varies by region (adaptive thresholding)

Otsu's Method (Optimal Global Threshold)

Maximizes inter-class variance between foreground and background: σ²_between(T) = w₀(T) × w₁(T) × [μ₀(T) - μ₁(T)]² w₀, w₁ = class probabilities (from histogram) μ₀, μ₁ = class means Equivalently: minimize intra-class variance. T* = argmax σ²_between(T)

Adaptive Thresholding

T(x,y) = mean of neighborhood - C (Mean Adaptive) T(x,y) = gaussian_weighted_mean - C (Gaussian Adaptive) C = constant (fine-tunes threshold, usually small positive) Use when: illumination varies across image (shadow, uneven lighting)
